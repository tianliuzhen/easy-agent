======================================
快速操作指南
======================================

## 第一步：执行数据库脚本 (必须)

1. 打开命令行，连接 MySQL:
   mysql -h localhost -P 3301 -u root -p123456

2. 选择数据库:
   use easy-agent;

3. 执行脚本:
   source F:\WorkSpace\MyGithub\easy-agent\ea-service\config\initsql\model_platform_table.sql;

4. 验证数据:
   SELECT * FROM ea_model_platform;

应该看到 4 条记录：deepseek, siliconflow, openai, ollama

## 第二步：启动后端服务

1. 进入服务目录:
   cd F:\WorkSpace\MyGithub\easy-agent\ea-service

2. 启动服务:
   mvn spring-boot:run

3. 测试接口 (新窗口):
   curl -X POST http://localhost:8080/eaAgent/modelPlatform/list
   
   或
   
   curl -X POST http://localhost:8080/eaAgent/ai/queryChatModelTypeList

## 第三步：前端集成 (可选)

如果需要在 frontend 中访问模型平台配置页面:

1. 在 App.tsx 中添加路由:
   <Route path="/pageTool/ModelPlatformConfig" element={<ModelPlatformConfig />} />

2. 或者在 AgentManager 中添加访问链接

3. 启动前端:
   cd ea-web
   npm run dev

## 常见问题

### Q1: 数据库连接失败？
A: 检查 MySQL 是否运行，端口是否为 3301

### Q2: Maven 构建失败？
A: 检查 pom.xml 依赖是否正确，Maven 仓库配置

### Q3: 接口返回空数据？
A: 检查数据库表是否有初始化数据

### Q4: 编译错误？
A: 运行 mvn clean compile 重新编译

## 回滚方案

如果需要恢复到枚举版本:

1. 删除表:
   DROP TABLE IF EXISTS ea_model_platform;

2. 恢复 EaAgentController.java:
   将 queryChatModelTypeList 方法改回:
   return BaseResult.buildSuc(ModelTypeEnum.getAll());

3. 删除相关类:
   - ModelPlatformService.java
   - ModelPlatformServiceImpl.java
   - ModelPlatformController.java
   - EaModelPlatformReq.java
   - EaModelPlatformResult.java
