2. 参考 [ToolExecutionAdvisor.java](ToolExecutionAdvisor.java) 新增一个滑动窗口的Advisor

    上下文轮数是比较暴力的限制方式，当达到轮数限制时，可能上下文还远远低于窗口限制
    因此还支持其他策略：滑动窗口和上下文压缩
    滑动窗口
    当每轮对话结束时，标记大模型input token和output token， 再下轮开始时进行计算当token 总数大于（窗口限制*阈值）触发裁剪动作，滑动窗口和论数概念类似，达到阈值限制进行丢弃最早的message，系统	会从前向后按轮的粒度进行裁剪，保留更近消息，直到token 小于当前限制
